# Campistry Link — mobile app shell

Native iOS + Android wrapper around the parent portal (`campistry_link_parent.html`
at the repo root), built with [Capacitor](https://capacitorjs.com). This folder
is a self-contained Node project — it doesn't touch the root `package.json` or
the rest of the (no-build-step) Campistry scheduler codebase.

**There is exactly one source of truth for the app's HTML/CSS/JS: the files at
the repo root.** This project never contains a hand-edited copy — `www/` is
generated by `scripts/sync-www.js` and is gitignored. Edit
`campistry_link_parent.html` / `campistry_link.css` / etc. at the repo root as
usual, then re-sync.

## Layout

- `capacitor.config.json` — app ID (`com.campistry.link`), display name, splash/status-bar config
- `scripts/sync-www.js` — copies the repo-root Link files into `www/`
- `resources/icon.png` (1024×1024), `resources/splash.png` (2732×2732) — master brand art
- `android/`, `ios/` — the native projects (committed; build output is gitignored)

## Setup (first time)

```
cd mobile/campistry-link
npm install
npm run sync        # populates www/, then `cap sync` copies it + plugins into android/ + ios/
```

## Day-to-day workflow

Whenever you change `campistry_link_parent.html`, `campistry_link.css`,
`campistry_notes_quick.js`, `campistry_link_capacitor.js`, `config.js`, or the
vendored `supabase-js@2.js` at the repo root:

```
npm run sync
```

Then open the native IDE and run/build as usual:

```
npm run open:android   # opens android/ in Android Studio
npm run open:ios       # opens ios/App/App.xcodeproj in Xcode (macOS only)
```

## What's already wired up

- **Material 3 UI** — the web app already targets this look; nothing app-shell-specific needed there.
- **Safe areas** — bottom nav / FAB respect `env(safe-area-inset-bottom)` for the iPhone home indicator.
- **App icon + splash** — generated from `resources/icon.png` / `resources/splash.png` (see "Regenerating icons" below). Swap those two master files for real brand art whenever you have it and regenerate.
- **Android hardware back button** — `campistry_link_capacitor.js` maps it to closing the open drawer/modal/detail view, then to in-app page history, then finally to exiting the app (via `@capacitor/app`'s `backButton` listener).
- **Deep links for parent invites** — the existing `?invite=TOKEN` web flow now also has a native path: `campistry_link_capacitor.js` listens for `appUrlOpen` and calls `window.__handleParentInviteToken(token)` (exposed by `campistry_link_parent.html`). Registered schemes:
  - Android: `com.campistry.link://` (auto-registered by Capacitor in `strings.xml` → `custom_url_scheme`)
  - iOS: `campistrylink://` (registered in `Info.plist` → `CFBundleURLTypes`)
  - **Not yet done:** wiring the camp's invite-email link generator (wherever `?invite=` links are actually created today) to *also* emit one of these custom-scheme URLs, or better, real Universal Links / Android App Links (see below) so a tapped link opens the app directly instead of a browser.
- **`target="_blank"` download links** (Forms/Camper Mail file downloads) — Capacitor's default WebView delegate opens these in the system browser automatically; not specifically tested on-device yet, flagged here as a first thing to check on a real build.

## What's NOT done yet (needs a real device/Mac/accounts — can't be done in this sandbox)

1. **Real brand art.** `resources/icon.png` / `resources/splash.png` are a placeholder mark (the Messages icon glyph on the app's brand green) rendered locally — there was no existing 1024×1024+ source art in the repo to work from. Swap in real artwork, then regenerate (see below).
2. **Universal Links (iOS) / App Links (Android)**, so invite links open the installed app directly instead of (or in addition to) a browser tab:
   - iOS: enable the "Associated Domains" capability in Xcode, add `applinks:link.campistry.app` (or whatever domain hosts invite links), and host `/.well-known/apple-app-site-association` on that domain.
   - Android: add an `<intent-filter>` with `android:autoVerify="true"` in `android/app/src/main/AndroidManifest.xml`, and host `/.well-known/assetlinks.json` on the domain, containing the app's release-signing SHA-256 fingerprint.
   - Until this is set up, the custom URL schemes above (`campistrylink://`, `com.campistry.link://`) work as a fallback *if* the invite link the camp sends is (or redirects through) one of those schemes.
3. **Signing & store accounts** — an Apple Developer Program membership ($99/yr) and Google Play Console account ($25 one-time), each app registered with the bundle ID `com.campistry.link` reserved on both platforms, and signing certificates/keystores set up (Xcode automatic signing, or a manual distribution certificate + provisioning profile for iOS; a release keystore for Android — **back this keystore up somewhere safe, losing it means you can never update the Play Store listing again**).
4. **Store listing assets** — screenshots (per required device sizes), a description, a support URL, and a **privacy policy URL** (mandatory on both stores — this app handles camper medical/emergency-contact data, so the policy needs to actually reflect that).
5. **Permissions justification** — neither store should need much from this app today (no camera/location/contacts usage found in the current code), but Apple's App Privacy questionnaire and Google's Data Safety form both need to be filled out honestly based on what the Supabase backend stores.
6. **Actually building** — `npx cap add ios`/`android` succeeded in this sandbox (scaffolding only — no Android SDK or Xcode/CocoaPods toolchain is installed here, so no build was attempted). You'll need Android Studio (for Android) and a Mac + Xcode (for iOS, non-negotiable — Apple requires it) to open these projects, resolve dependencies, and produce a signed build.
7. **Push notifications**, if wanted later — would need `@capacitor/push-notifications` plus APNs (Apple) / FCM (Google) setup; nothing here today.

## Regenerating icons/splash from new master art

This sandbox couldn't install `@capacitor/assets` (its `sharp` dependency needs
a binary download the sandbox's network policy blocks), so the current
icon/splash files across `android/app/src/main/res/**` and
`ios/App/App/Assets.xcassets/**` were rendered directly at each required
resolution via a one-off Playwright script rather than resized from a single
master. On a machine with normal internet access, once you have real branding
art, the supported path is:

```
npm install --save-dev @capacitor/assets
# replace resources/icon.png (1024x1024) and resources/splash.png (2732x2732)
npm run assets   # regenerates every density/size for both platforms from those two files
npm run sync
```

## Bundle ID / app name

Currently `com.campistry.link` / "Campistry Link" (set in `capacitor.config.json`,
and already propagated into `android/app/src/main/res/values/strings.xml` and
